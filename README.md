# Custom Yocto Project for STM32MP1 Odyssey (with Raspberry Pi DSI Display)

This project uses **kas** to completely automate environment setup, layer fetching, and building, replacing manual `repo` syncs and interactive scripts.

## Prerequisites

Make sure you have **Docker** installed on your host machine and your user is added to the `docker` group.

## 1. Clone This Project

Clone this repository (which contains your custom configuration and display layer) to your local machine:

```bash
git clone <YOUR-GIT-REPOSITORY-URL> my-yocto-project
cd my-yocto-project
```

## 2. Get the kas-container Script

Download the official `kas-container` script, which automates running the build inside a verified Docker environment [🌐](https://readthedocs.io):

```bash
wget https://githubusercontent.com
chmod +x kas-container
```

## 3. Build the Project

Run the build. `kas` will automatically clone the STMicroelectronics manifest, pull the `meta-st-odyssey` layer, inject the EULA acceptance, apply your kernel patches, and start the compilation [🌐]:

```bash
./kas-container build kas-project.yml
```

## 4. Create the SD Card Image

Once the build finishes successfully, navigate to the deployment directory and generate the bootable SD card image using the ST flash layout script:

```bash
# Move to the build output directory generated by kas
cd build/tmp-glibc/deploy/images/stm32mp1

# Create the bootable raw image
./scripts/create_sdcard_from_flashlayout.sh flashlayout_st-image-weston/optee/FlashLayout_sdcard_stm32mp157c-odyssey-optee.tsv
```
